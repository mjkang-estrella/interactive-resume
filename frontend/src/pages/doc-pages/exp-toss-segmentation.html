Customer Segmentation & Personalization

Background

The Joint Account product served a wide range of users from couples to hobby clubs, but all were being treated as a single user group. To increase relevance and satisfaction, I initiated a project to personalize user experiences through behavioral segmentation and modular feature design.

Problem

The challenge was how to segment users meaningfully without overcomplicating product management. Creating too many variants would fragment resources, yet treating all users identically limited engagement and satisfaction.

Solution

  1. Data-Driven Segmentation:
    • Analyzed over 1 million user behaviors using SQL and Python to identify distinct usage patterns.
    • Segmented users into Couples, Partners, Clubs & Activity Groups, and Others, focusing only on cases with clearly differentiated behaviors.
  2. Personalized Feature Design:
    • Built modular landing pages and functions tailored to each group’s needs — e.g., budgeting tools for couples, shared asset tracking for clubs, and message boards for group coordination.
    • Designed these features as modular components, enabling efficient iteration without heavy maintenance costs.
  3. Qualitative Validation:
    • Conducted user interviews and feedback sessions to ensure each segment’s design accurately reflected its core motivations and use cases.

Result & Analytics

The initiative increased NPS by 20 points and daily joint account openings by 30%. The segmentation framework became a model for designing scalable personalization, balancing user diversity with operational efficiency.
