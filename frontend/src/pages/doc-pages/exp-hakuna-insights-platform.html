Data Platform

Background

When I worked on the team, all data analysts had recently left, leaving a major operational gap. Teams relied on ad hoc SQL queries on BigQuery, and I became the main contact for data requests across departments. However, inconsistent query logic, fragmented data sources, and lack of centralized policies caused confusion and inefficiency in product decision-making.

Problem

Through discussions with product, engineering, and business teams, I identified three core issues:

  1. No data collection policy: Developers were logging events with inconsistent formats and naming conventions, making data unreliable.
  2. No single source of truth: Key metrics such as “Active User” or “Retention” were defined differently across teams, leading to misaligned reports and duplicated analyses.
  3. No centralized dashboards: Teams lacked visibility into live KPIs and had to run manual queries repeatedly for updates, delaying decisions.

These issues resulted in slow turnaround times, inconsistent insights, and limited trust in data.

Solution

  1. Defined and implemented data collection policies:
    • Standardized event-logging structure and naming conventions across teams.
    • Required PMs and engineers to sync on logging schema before each feature release.
  2. Established unified metric definitions:
    • Collaborated with strategy and business teams to create shared definitions for all core KPIs.
    • Published a single source of truth document and trained PMs to apply it consistently.
  3. Centralized data visualization:
    • Learned Tableau from a neighboring team and independently built real-time dashboards for key metrics like MAU, retention, and session engagement.
    • Linked all fragmented data pipelines into one integrated BigQuery dataset accessible to all product teams.

Result & Analytics

  • Reduced data turnaround time by 40%, enabling faster experimentation and decision cycles.
  • Improved data accuracy and trust across the organization by enforcing unified definitions.
  • The dashboards became the default reference point for product and leadership meetings, removing redundant SQL queries and manual reporting.
  • The project created a foundation for scalable, data-driven operations — turning a data bottleneck into a self-service analytics culture.
